[React] destructure parameter

When I wrote in reactjs, I came across a problem like this:

TypeError: setSearchInput is not a function

The code of child component is like this: 
      
    const SearchInput = (searchInput, setSearchInput, setWord, setMeaning) => {
        // doing something
    };
The code of parent component is like this: 
      
    <SearchInput
        searchInput={searchInput}
        setSearchInput={setSearchInput}
        setMeaning={setMeaning}
        setWord={setWord}
    />
After I searched online, I found that the original way to receive the properties is like this. const SearchInput = (props) => {} In this way, the functional component receives the prop, an object, as the first argument. And when I want to access the argument I passed in, I would call it like {props.searchInput}.
But sometimes it is better to destructure the object to make it clear what parameters are passed as properties. In this way, I destructure the object by wrapping all of the items in curly brackets, like this 
        
    const SearchInput = ({searchInput, setSearchInput, setWord, setMeaning}) => { 
        // doing something 
    };
So when I call it, I can directly call it by the name, like {searchInput}.
