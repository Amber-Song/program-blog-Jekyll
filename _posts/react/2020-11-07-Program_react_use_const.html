[React] why using const keyword in reactjs

In vanilla js,
  • variable with const keyword can not be reassigned and it is block-scoped
  • variable with let keyword is reassigned and it is block-scoped
  • variable with var keyword is also reassigned but it is function-scoped
That is why in vanilla js, we usually use the let keyword. So I was confused why in all the tutorial, they wrote variables with const keyword in useState hook even when I was pretty sure that those variables would change in the following commend. Then I found that when properties or states change, the component will be rerendered. And the function will be executed again which will cause it to create a new scope again. Then, the variable with const will be assigned a new value instead of reassigning values.
